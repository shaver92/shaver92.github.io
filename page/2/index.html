一句话总览

1 内容结构与重点

1.1 章节脉络

  • §1 开场：通⽤智能体应⽤步⼊⽣产⼒阶段
  • §2 概念起源：从 Prompt Engineering 到 Harness Engineering
  • 2.1 三个时代的嵌套关系
  • 2.2 Karpathy 推⽂与 Context Engineering 的兴起
  • 2.3 Hashimoto 的正式命名与 DevOps 类⽐
  • 2.4 ⾥程碑时间线
  • §3 四⼤⽀柱：Harness Engineering 的理论内核
  • 3.1 ⽀柱⼀：代码库即真相源（Codebase as Source of Truth）
  • 3.2 ⽀柱⼆：机械化架构约束（Mechanized Architectural Constraints）
  • 3.3 ⽀柱三：反馈循环（Feedback Loops）
  • 3.4 ⽀柱四：熵管理（Entropy Management）
  • 3.5 核⼼公式与震撼数据

1.2 核心收获

  • 形成了从“理解概念”到“可执行落地”的完整路径。
  • 给出了配置、验证、排障的可操作检查点。
  • 可直接沉淀为团队 SOP 或个人复盘清单。

2 关键结论

  • 先搭框架再追求高级能力：优先打通主链路，再做性能与扩展。
  • 诊断与日志是稳定运行的基础能力，应和功能开发同等重视。
  • 版本、模型、技能与通道配置需要“变更即验证”的操作习惯。

云端部署

ArkClaw / KimiClaw / MaxClaw
类似deepseek客户段，云端厂商已经集成在自己app中，可以开箱即用，无需安装

本地安装

QClaw / AutoClaw / Molili
需要本地安装app，但是安装成本低，只需要傻瓜式点击下一步即可

开源自部署

openclaw-cn / LinClaw / CoPaw
需要下载源码以及对应依赖，再进行部署，有一定上手难度
