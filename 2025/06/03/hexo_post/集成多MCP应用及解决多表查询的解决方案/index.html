集成多 MCP 应用及解决多表查询的解决方案

目标

  • 集成多MCP服务，包括第三方MCP和本地MCP
  • 自然语言转多表查询效果不佳的一些想法（重点，见 数据库表的定制查询）

https://github.com/shaver92/easyworktool.git

本地sqlite的MCP服务

这个是用sqlite的操作封装的MCP服务，很有趣的是，这个用来创建库表，包括进行造数据操作，都很精准无误，

下方是一些操作截图：

有一些思考：

  • 现在支持的都是单步操作，比如建表和造数据，都只能一张表操作完才能操作下一张
  • 可能是对应工具封装没有批量操作的
  • 但是大模型已经将下一步操作的命令准备好了，感觉对于大模型交互，能够编排好流程，充分利用大模型的输出还是有很大的提升

数据库表的定制查询

现在自然语言转多表查询，很难让大语言模型去完全理解库中的所有表和字段的关系
现在我想是不是可以将所有的日常查询做成分类，然后封装到MCP对应独立的工具中，比如
商品查询 -> 独立一个工具方法
人员查询 -> 独立一个工具方法
库存查询 -> 独立一个工具方法

上面的查询，会有独立的查询数据库的sql，对应sql需要包含所有当前能够想到的条件，这样就可以自然语言 到具体分类到对应的sql查询，这样就规避了让大模型自己需要将用户问题直接转成多表查询的sql不准确的问题

可以见flexible_query_mcpserver.py中 就是将订单查询 和 对应用户查询分开

后续出现新的查询场景，可以直接增加对应的工具方法或是MCP服务都可以
