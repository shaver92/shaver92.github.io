1 简要描述

上次我们熟悉了Harness Engineering的实现之一Gstack，并使用

安装到了我们claude code中可以进行使用，现在我们将使用如下流程进行效果测试

Think Plan Build Review Test Ship Reflect
/office-hours → /autoplan → 写代码 → /review → /qa → /ship → /retro

2 实现步骤

我们现在已实现一个日常记账本的功能为例说明上面的步骤

2.1 /office-hours

我想做一个日常记账本，来统计家庭费用支出
1 提供不同身份登录，比如爸爸，妈妈，女儿，儿子，爷爷，奶奶，外公，外婆等
2 支持指出费用类型自定义
3 支持各种维度统计，比如周 月 年
4 支持个人设置自己支出上限，并提供预警信息
5 目前我现在想到的是用streamlit进行实现，方便部署和线上使用，看这个是否合适，或是你有比较好的想法

2.1.1 生成CLAUDE.md

2.2.2 开始进行对需求多轮询问

开始进行需求的审核是否合理

生成最终产物 md

2.2 /autoplan

执行/autoplan 生成最小可实现 可测试plan

生成最终的plan

2.3 生成代码

开始生成对应代码 会自动按照拆分的任务进行实现

第 4 步（代码写完了）

/review
重点查：权限绕过（未登录访问分享链接）、XSS（恶意 Markdown 内容）、
链接过期逻辑是否生效。

第 5 步

/qa https://staging.mynotesapp.com
测试路径：

  1. 创建笔记 → 点分享 → 复制链接 → 无痕窗口打开（验证可访问）
  2. 设置密码保护 → 无痕窗口打开（验证需要密码）
  3. 过期链接访问（验证显示过期提示）
  4. 分享一个空笔记
  5. 分享一个包含代码块和表格的长笔记

第 6 步
